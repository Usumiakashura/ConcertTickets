Техническое задание для 1й части стажировки

Система бронирования билетов на концерт

Есть 2 роли - пользователь и админ. Пользователь может купить билет, админ может добавить концерт.

Концерт минимально имеет:

Исполнителя (группу)
Количество билетов
Дату
Место, где проходит концерт.
Тип концерта (классическая музыка, вечеринка, опэнэйр). Для классической музыки нужно отдельно иметь информацию о типе голоса солиста (альт, тенор, и т.д) и название концерта, композитора. Для опэнэйра – как проехать и хэдлайнер. Для вечеринке – возрастной ценз. (к примеру, суть в том, что есть как общие части, так и отличные, нужно использовать наследование и подумать как хранить в бд иерархические структуры)

Пользователь может открыть карту и увидеть где какой концерт проходит. Вверху страницы поиск для того, чтобы отфильтровать концерты по названию а так же по интересующему типу. Плюс он может забронировать билет. Бронируя билет получает конфирмейшн на почту.

Получение данных о пользователе по протоколу OpenID

Бонусные пункты:

а) Замена бронирования на покупки. Покупка происходит через пайпал. Тестовый аккаунт будет предоставлен.
б) Возможность добавления промокода со скидкой.

Сам тестовый проект представляет собой простое веб приложение где есть минимальная клиентская часть, авторизация, операции работы с бд и простенькая бизнес логика. Стэк - ( .NET Core, ASP.NET Core, EF Core / или Python/Django ), Vue.JS или ReactJS для фронта
